
# Full Stack Java Interview Questions (Organized)

**Note:** Some topics/questions mentioned were repeated or phrased differently in the raw notes. This list consolidates and organizes them for easier review and preparation. Also, there are questions listed here for technologies/topics not covered during training; they have been included to provide context for the kind of questions you can expect to be asked, but always keep in mind that a topic being listed here does not guarantee you will be asked about it. In a similar vein, a topic/question not being present here does not indicate you will not be asked about it. For instance, Docker is not mentioned here, but it is a topic covered in training, so you may be asked about it in the interview. This data was gathered by getting feedback from associates who went through interviews, so this is NOT an exhaustive collection of all questions asked.

---

## 1. Core Java
- Differences: Inheritance vs Polymorphism
- Abstract class vs Interface
- Data structures: List, Set, Map (when to use each)
- ArrayList vs LinkedList
- HashMap vs ConcurrentHashMap (null keys, thread safety)
- Comparable vs Comparator
- Singleton design pattern (usage, thread safety)
- Design patterns: Singleton, Factory, others (explain them)
- Exception hierarchy
- Collection hierarchy 
- Adding/removing Integers in a List (autoboxing, remove behavior)
- Switch case with default statement
- Basic Java questions (general knowledge)

## 2. Spring & Spring Boot
- Key annotations: @RestController, @Service, @Component, @SpringBootApplication, @Configuration, @Scope
- Dependency injection (how it works)
- Application configuration (XML, @Configuration, properties file)
- Difference between @Service and @Component
- Spring vs Spring Boot
- Spring Security (how it works, securing APIs, JWT automation)
- List annotations used in your project

## 3. Microservices
- How microservices communicate (REST APIs, message queues, service mesh, shared DB, SOAP)
- Layers in backend architecture
- Is Controller thread-safe?

## 4. Databases & SQL
- Databases used
- SQL joins
- Primary key vs Unique key vs regular constraint
- Can UNIQUE columns contain NULL values?
- Stored procedure vs function (can a procedure be used in a function?)
- SQL query writing

## 5. Coding Challenges
- Longest common prefix
- Shift an array by k places
- Alphabetize an array of strings
- Group a list of strings by their length and count how many strings fall into each group
- Create a Stack that can return the minimum value
- Sort a list of employees by department, then by ID

## 6. Frontend (React & Angular)
- ReactJS: hooks, prop drilling, class components, Redux
- Angular: modules, components, services, decorators, data binding, state sharing between siblings, performance, libraries used
- Difference between Angular forms and React forms
- How to make API calls from frontend to backend
- How to cache/prevent duplicate API calls
- Authentication (JWT in cookie, storing user ID, UserAuth service)
- RBAC: route guards in Angular, backend endpoint annotations
- Directives used in UI
- DOM knowledge

## 7. System Design & Architecture
- System design basics
- Set vs ArrayList vs LinkedList
- Full stack development and microservices
- General knowledge: What is idempotent?

## 8. Miscellaneous
- How did you use AI in your project?
- Experience with AWS, Kafka
- General technical and behavioral questions

# Previous Categories Assessed
**NOTE**: these are general categories panelists have used to assess candidates in the past. 

**Core Java:**
Assessment covers hands-on experience with OOP principles, exception handling, collections, and multithreading. Candidates are expected to write clean, efficient, and reusable code.

**Java + Spring Boot:**
Assessment focuses on the ability to develop RESTful APIs using Java and Spring Boot, understanding of annotations, configuration management, and application deployment.

**Git:**
Assessment includes proficiency in version control using Git, including branching, merging, repository maintenance, and exposure to CI/CD tools.

**Spring Framework:**
Assessment covers knowledge of core Spring concepts such as dependency injection, inversion of control, and application context, as well as configuring and integrating Spring modules.

**SQL**
Assessment includes writing efficient SQL queries, joins, and stored procedures, with consideration for query optimization and database performance tuning.

**SDLC Methodologies:**
Assessment focuses on participation in Agile ceremonies and sprint tasks, estimation, planning, and collaboration across sprints.

**Agile:**
Assessment covers effectiveness in Agile sprints, contribution to planning, demos, retrospectives, and timely delivery of assigned stories.

**NOTE:** I wasn't able to find any data on assessments made for client-side technologies, but seeing as we have documentation on React/Angular questions being asked assume you will be asked about working with Angular/Typescript.
